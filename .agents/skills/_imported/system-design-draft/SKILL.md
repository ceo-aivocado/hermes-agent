# System Design Draft

For architecture or research tasks, produce:

1. Problem statement.
2. Constraints and non-goals.
3. Options considered.
4. Recommended path.
5. Risks and mitigations.
6. Implementation phases.
7. Verification plan.

Keep output suitable for `docs/` or a GitHub issue. Do not turn research into production changes without MAIN.
